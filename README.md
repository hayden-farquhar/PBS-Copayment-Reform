# PBS Copayment Reform: Replication Code

Analytical code for: **"Did Australia's 2023 PBS copayment reduction increase prescription volumes or reduce geographic inequity? A natural experiment using methodological triangulation"**

**Author:** Hayden Farquhar, MBBS MPHTM
**Contact:** hayden.farquhar@icloud.com
**ORCID:** 0009-0002-6226-440X

## Overview

This repository contains the data acquisition, cleaning, and analysis scripts to replicate all findings reported in the manuscript. The study evaluates Australia's January 2023 reduction in the Pharmaceutical Benefits Scheme (PBS) general patient copayment from $42.50 to $30.00, using interrupted time series, Bayesian structural time series (CausalImpact), and difference-in-differences methods across 284 months of national dispensing data.

## Replication Instructions

Scripts are numbered sequentially and should be run in order from the project root directory.

### Step 1: Data Acquisition

```r
source("scripts/01_data_acquisition.R")
```

This downloads ABS data files (SEIFA, Remoteness Areas, LGA shapefiles, ERP) automatically. **AIHW PBS data requires manual download** — the script prints instructions:

1. Go to https://www.aihw.gov.au/reports/medicines/pbs-monthly-data/data
2. Download all supplementary Excel workbooks (ATC Level 1 and Level 2, historical and current)
3. Save `.xlsx` files to `data/raw/`

Expected files in `data/raw/` after this step:
- `AIHW-HWE-098-PBS-ATC1-prescriptions-historical-data.xlsx`
- `AIHW-HWE-098-PBS-ATC1-prescriptions-monthly-data.xlsx`
- `AIHW-HWE-098-PBS-ATC2-prescriptions-historical-data.xlsx`
- `AIHW-HWE-098-PBS-ATC2-prescriptions-monthly-data.xlsx`
- `SEIFA_2021_LGA.xlsx`
- `RA_2021_AUST.xlsx`
- `LGA_2021_AUST_allocation.xlsx`
- `MB_2021_AUST.xlsx`
- `ERP_LGA_2001_2024.xlsx`

### Step 2: Data Cleaning

```bash
source .venv/bin/activate
python scripts/02_data_cleaning.py
```

Processes raw Excel workbooks into tidy monthly panel CSVs:
- `data/processed/pbs_national_monthly.csv` (~8 MB)
- `data/processed/pbs_state_monthly.csv` (~56 MB)
- `data/processed/pbs_lga_monthly.csv` (~2.9 GB)
- `data/processed/lga_characteristics.csv`
- `data/processed/erp_lga.csv`

### Step 3: Descriptive Analysis

```r
source("scripts/01b_descriptive_analysis.R")
```

Produces descriptive time series plots and summary statistics (Table 1 inputs, Figure 1).

### Step 4: Interrupted Time Series

```r
source("scripts/03_its_analysis.R")
```

Segmented regression with Newey-West HAC standard errors across 86 ATC Level 2 classes. Produces ITS results tables and forest plots.

### Step 5: Bayesian Causal Impact (Primary Analysis)

```r
source("scripts/04_causal_impact.R")
```

CausalImpact models using concessional (65+) prescriptions as a control series. Produces the primary causal effect estimate and ATC-level/state-level stratifications.

### Step 6: Equity Analysis

```r
source("scripts/05_equity_analysis.R")
```

Difference-in-differences panel regression by SEIFA quintile and remoteness category, triple-difference test, equity impact ratios, spatial autocorrelation (Moran's I), and choropleth maps.

### Step 7: Sensitivity Analyses

```r
source("scripts/07_sensitivity.R")
```

Six robustness scenarios: 60-day dispensing adjustment, COVID-19 exclusion, placebo intervention dates, shortened pre-period, control correlation filtering, and alternative equity windows.

## Notes

- R scripts use `here::here()` for path resolution. Open the project as an RStudio project or ensure `.git` is in the root directory.
- The Python script resolves paths relative to its own location (`scripts/` → project root).
- The LGA-level processed CSV (`pbs_lga_monthly.csv`, ~2.9 GB) is not tracked by git due to file size. It is regenerated by `02_data_cleaning.py`.
- Runtime for the full pipeline is approximately 2-3 hours, with the majority spent on CausalImpact MCMC fitting (Step 5) and the equity bootstrap (Step 6).
